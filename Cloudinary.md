# Railway-deployed Django app for Image Uploads
- [x] Get your Cloudinary credentials
    - In the Cloudinary Console, go to Settings → API Keys and copy your Cloud name, API Key, and API Secret (never expose the secret).
- [x] Install and configure in Django
    - ```pip install cloudinary```
    - In settings.py, add cloudinary to INSTALLED_APPS and import/configure Cloudinary (use environment variables in production).
- [x] Update your model and form
    - In models.py, replace ImageField with CloudinaryField('image') so uploads go to the cloud and the delivery URL is saved in your DB.
    - In forms.py, use CloudinaryFileField for the image field.
- [ ] Deploy to Railway
    - Add your Cloudinary env vars in Railway (Cloud name, API Key, API Secret) and deploy your app as usual.
